# تطبيق إشارات الخيارات الثنائية - ملخص المشروع

## ✅ التنفيذ المكتمل

تطبيق جاهز للإنتاج "إشارات فقط" للهاتف المحمول مع دعم العربية RTL كأولوية، واجهة داكنة جريئة، وبنية خلفية كاملة.

## 📁 هيكل المشروع

```
botsignal/
├── apps/
│   ├── api_nest/              # خادم NestJS
│   │   ├── src/
│   │   │   ├── auth/          # مصادقة قائمة على الجهاز
│   │   │   ├── verify/        # التحقق من الحساب
│   │   │   ├── postback/      # استقبال postback الشريك
│   │   │   ├── signals/       # إدارة الإشارات
│   │   │   ├── admin/         # نقاط نهاية Admin
│   │   │   ├── platforms/     # معلومات المنصة
│   │   │   ├── fcm/           # الإشعارات الفورية
│   │   │   └── prisma/        # خدمة قاعدة البيانات
│   │   └── prisma/
│   │       ├── schema.prisma  # مخطط قاعدة البيانات
│   │       └── seed.ts        # بيانات البذور
│   └── mobile_flutter/        # تطبيق Flutter للهاتف المحمول
│       └── lib/
│           ├── core/          # الخدمات الأساسية، التصميم، التوجيه
│           └── features/      # شاشات الميزات
├── docker-compose.yml          # إعداد PostgreSQL
└── SETUP.md                   # دليل الإعداد الكامل
```

- ✅ واجهة برمجة تطبيقات التحقق من الحساب (3 حالات: VERIFIED_ACTIVE، VERIFIED_NO_DEPOSIT، NOT_UNDER_TEAM)
- ✅ نقاط نهاية استقبال postback لـ Quotex & Pocket Option
- ✅ إدارة الإشارات مع محفزات الإشعارات الفورية
- ✅ نقاط نهاية Admin (محمية بمفتاح API)
- ✅ خدمة الإشعارات الفورية FCM
- ✅ وثائق Swagger API
- ✅ تحديد معدل على نقطة التحقق
- ✅ PostgreSQL مع Prisma ORM
- ✅ بيانات البذور مع حسابات تجريبية

### تطبيق الهاتف المحمول (Flutter)

- ✅ شاشة اختيار المنصة
- ✅ تدفق التحقق من الحساب
- ✅ شاشة نتيجة التحقق مع 3 حالات
- ✅ خلاصة الإشارات (مقفلة/مفتوحة حسب الحالة)
- ✅ شاشة تفاصيل الإشارة
- ✅ حاسبة المخاطر
- ✅ الإعدادات (تبديل اللغة، تغيير المنصة، تسجيل الخروج)

- مصادقة الجهاز القائمة على JWT
- حماية مفتاح API للمسؤول
- التحقق من سر postback

## 📊 مخطط قاعدة البيانات

- **devices**: تسجيل الجهاز ورمز FCM
- **accounts**: تتبع حالة حساب الوسيط
- **device_accounts**: ربط الجهاز-الحساب متعدد
- **postback_events**: سجل تدقيق لجميع postbacks
- **signals**: بيانات الإشارة مع حالة الأخبار
- **notifications_log**: سجلات تسليم الإشعارات الفورية

## 🚀 البدء السريع

1. **تشغيل قاعدة البيانات:**
   ```bash
   docker-compose up -d postgres
   ```

2. **إعداد الخادم:**
   ```bash
   cd apps/api_nest
   npm install
   cp env.example .env
   # عدّل .env
   npm run prisma:generate
   npm run prisma:migrate
   npm run prisma:seed
   npm run start:dev
   ```

3. **إعداد Flutter:**
   ```bash
   cd apps/mobile_flutter
   flutter pub get
   flutter run
   ```

راجع `SETUP_AR.md` للحصول على تعليمات مفصلة.

## 🧪 الحسابات التجريبية

حسابات تجريبية مملوءة:
- `TEST_ACTIVE_123` (QUOTEX) → VERIFIED_ACTIVE
- `TEST_NO_DEPOSIT_456` (QUOTEX) → VERIFIED_NO_DEPOSIT

## 📝 نقاط نهاية API

### عام
- `POST /api/auth/register` - تسجيل الجهاز
- `POST /api/verify` - التحقق من الحساب
- `GET /api/platforms` - الحصول على المنصات

### مصادق
- `GET /api/signals` - الحصول على الإشارات (يتطلب verified active)
- `POST /api/auth/fcm-token` - تحديث رمز FCM

### Admin
- `POST /api/admin/signals` - إنشاء إشارة

### Postback
- `GET /api/postback/quotex` - postback Quotex
- `GET /api/postback/pocket-option` - postback Pocket Option

## 🎨 ميزات UI/UX

- **عربي أولاً**: تخطيط RTL، العربية كلغة افتراضية
- **تصميم داكن**: خلفية سوداء (#000000) مع لهجات خضراء/زرقاء نيون
- **طباعة جريئة**: خطوط كبيرة، قابلة للقراءة
- **CTAs واضحة**: أزرار بارزة للإجراءات
- **حالات فارغة**: رسائل مفيدة عند عدم وجود بيانات
- **حالات التحميل**: دوارات أثناء العمليات غير المتزامنة
- **معالجة الأخطاء**: رسائل خطأ سهلة الاستخدام

## 📱 تدفق تطبيق الهاتف المحمول

1. **Splash** → التحقق من رمز المصادقة
2. **اختيار المنصة** → اختر Quotex أو Pocket Option
3. **التحقق من الحساب** → أدخل معرف الحساب
4. **نتيجة التحقق** → عرض الحالة + CTA المناسب
5. **الرئيسية** → خلاصة الإشارات (إذا كان VERIFIED_ACTIVE)
6. **تفاصيل الإشارة** → عرض الإشارة + فتح الوسيط
7. **حاسبة المخاطر** → حساب حجم المركز
8. **الإعدادات** → اللغة، المنصة، تسجيل الخروج

## ⚠️ ملاحظات مهمة

- **لا تنفيذ تداول**: التطبيق يرسل إشارات فقط
- **صياغة محايدة**: يستخدم "إشارات"، "تحليل"، "مخاطر" (لا وعود بأرباح)
- **تتبع الإحالة**: يجب على المستخدمين التسجيل تحت فريقك
- **التحقق من postback**: التحقق من الخادم إلى الخادم للتسجيل + الإيداعات
